# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Afro Fashion Mobile seriously. If you believe you have found a security vulnerability, please report it to us as described below.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to the maintainers at [raphael@kirovdynamics.com](mailto:raphael@kirovdynamics.com) or open a private vulnerability report at [github.com/Raphasha27/afro_fashion_mobile/security/advisories](https://github.com/Raphasha27/afro_fashion_mobile/security/advisories).

You should receive a response within 48 hours. If you do not, please follow up to ensure your message was received.

### What to Include

- Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue, including how an attacker might exploit it

### Preferred Languages

We prefer all communications to be in English.

## Disclosure Policy

- The vulnerability report is received and assigned a primary handler.
- The report is confirmed and a list of affected products/releases is determined.
- The fix is prepared and tested.
- A security advisory is published and a patch is released.
- Once the fix is publicly available, the vulnerability is disclosed.

## Comments on this Policy

If you have suggestions on how this process could be improved, please submit a pull request or open an issue.
